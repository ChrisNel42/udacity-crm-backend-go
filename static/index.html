API Documentation

Introduction

Welcome to the API documentation for my CRM Backend. This page will guide you on how to use my API to access and interact with the customer DB.

Getting Started

To get started, you will need to execute the application on your machine. Please follow the steps in the README

API Endpoints

Below are the available API endpoints:

  • /customers GET - List all customers currently present in the DB
  • /customers/{id} GET - Get the customer with the specified id from the DB
  • /customers POST - Add a customer to the DB; Customer is sent in the body as JSON
  • /customers/{id} PUT - Update an existing customer in the DB; Customer is sent in the body as JSON - Field "id" has no effect
  • /customers/{id} DELETE - Delete an existing customer from the DB

Data structure

Below is an example of a valid JSON customer object:

        
        {
            "id": 1,
            "name": "John",
            "phone": "Doe",
            "email": "email@mail.com",
            "contacted": false
        }
  • id - Type int
  • name - Type string
  • phone - Type string
  • email - Type string
  • contacted - Type boolean

Authentication

All API requests require no authentication

Response Format

The API responses are returned in JSON format. The structure of the response will vary depending on the endpoint.

Error Handling

If an error occurs, the API will return an appropriate HTTP status code along with an error message in the response body, if it makes sense.
